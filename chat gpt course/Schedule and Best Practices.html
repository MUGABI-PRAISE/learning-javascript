Module 1: Introduction to JavaScript

1. What is JavaScript?

JavaScript is a powerful, high-level programming language mainly used to make web pages interactive.

  • HTML gives structure to a webpage.

  • CSS styles the webpage (colors, layout, fonts).

  • JavaScript adds life to a webpage (animations, dynamic content, user interactions).

Think of it like this:

If a webpage was a human,

  • HTML would be the bones and muscles.

  • CSS would be the clothes and hairstyle.

  • JavaScript would be the brain and movement.

A few facts about JavaScript:

  • Invented in 1995 by Brendan Eich in just 10 days!

  • Runs inside the browser (like Chrome, Firefox, Edge).

  • Can now run outside browsers too (on servers) using Node.js.

  • Modern websites (Facebook, YouTube, Twitter) use a lot of JavaScript.

Summary:
JavaScript is the language that makes websites dynamic and interactive.

2. How JavaScript Runs in the Browser

Every browser (Chrome, Firefox, Safari, etc.) has a JavaScript Engine a program that reads and executes JavaScript code.

BrowserJavaScript Engine
ChromeV8
FirefoxSpiderMonkey
SafariJavaScriptCore
EdgeChakra (Old Edge), V8 (New Edge)

How it works:

  • You write JS code.

  • The JS Engine inside the browser reads your code, translates it into machine code (0s and 1s), and executes it.

  • That's why you can interact with websites (click buttons, see animations, fill forms) all thanks to JavaScript running live in your browser.

Behind the Scenes (simple version):

  • Parse: Break code into understandable parts.

  • Compile: Translate code into machine code.

  • Execute: Run the machine code.

  • Optimize: Make your code run faster.

Modern browsers optimize JavaScript a lot behind the scenes so that websites feel fast and smooth.

3. Linking JavaScript to HTML

To use JavaScript in a webpage, you must link or embed it inside your HTML file.

Three ways to use JavaScript:

1. Inline JavaScript

(inside an HTML element)

html
<button onclick="alert('Hello World')">Click Me</button>

  • Good for very small scripts, but not recommended for serious development.

2. Internal JavaScript

(inside a <script> tag within your HTML)

html
<!DOCTYPE html>
<html>
<head>
  <title>My First JS Page</title>
</head>
<body>

<h1>Hello from HTML</h1>

<script>
  alert('Hello World from inside the page!');
</script>

</body>
</html>

  • Good for small experiments.

  • Code sits inside the HTML file.

3. External JavaScript File

(best method)

html
<!DOCTYPE html>
<html>
<head>
  <title>External JS Example</title>
  <script src="script.js" defer></script>
</head>
<body>

<h1>Hello from External JS!</h1>

</body>
</html>

  • The actual JavaScript code goes inside script.js (a separate file).

  • Cleaner and easier to manage big projects.

  • Always use defer for better performance (explained next).

Understanding defer and async

When you link a JavaScript file, you can use attributes like defer or async:

AttributeBehavior
deferWaits until HTML is fully loaded, then runs JS in order. (Recommended for most cases)
asyncRuns JS immediately once downloaded, without waiting for HTML. (Can cause problems if not careful.)

Best practice:

html
<script src="script.js" defer></script>

Always use defer so your JavaScript doesnt block the loading of your webpage.

4. console.log() and Debugging Basics

What is console.log()?

Its a built-in JavaScript function that prints information to the browsers console (not the webpage!).

Example:

javascript
console.log("Hello from JavaScript!");

  • It helps you see what's happening inside your code.

  • It's your best friend for finding bugs.

How to Open the Browser Console:

  • Chrome: Right-click Inspect Console Tab.

  • Firefox: Right-click Inspect Element Console Tab.

  • Shortcut: Press Ctrl + Shift + J (Windows) or Cmd + Option + J (Mac).

Why Debugging Matters:

  • Real-world developers don't write perfect code on the first try.

  • You will make mistakes.

  • Debugging helps you find and fix them by:

    • Checking values with console.log().

    • Reading error messages.

    • Understanding the flow of your code.

Module 2: Basics of JavaScript Syntax

1. Variables: var, let, and const

What is a variable?

  • A variable is like a container where you store data (like names, numbers, etc.).

  • You can create variables and then use them later in your program.

Three ways to create variables:

SyntaxBehavior
varOld way (before 2015). Has some problems (like weird behavior in scopes).
letNewer, better way. Use when you expect the value to change.
constNewer, better way. Use when the value should not change (constant).

Examples:

javascript
var myName = "David";    // old way
let age = 25;            // modern way (use this)
const country = "Uganda"; // modern way for values that don't change

Important Differences:

Featurevarletconst
ScopeFunction-scopedBlock-scopedBlock-scoped
Can reassign?YesYesNo
Can redeclare?YesNoNo

Best Practice:

Always use let and const.
Avoid var unless theres a strong reason (which is rare).

2. Data Types in JavaScript

Data type = the kind of value stored inside a variable.

JavaScript has 8 basic (primitive) data types:

TypeExample
String"Hello", "123", 'JavaScript'
Number25, 3.14, -100
Booleantrue, false
Nullnull Means "nothing" on purpose
UndefinedA variable that has been declared but not given a value
ObjectA collection of key-value pairs ({name: "David", age: 25})
SymbolUnique identifier (used rarely for special advanced tasks)
BigIntVery large numbers (not used often for now)

Examples:

javascript
let name = "Alice";         // String
let age = 30;               // Number
let isStudent = true;       // Boolean
let emptyValue = null;      // Null
let notAssigned;            // Undefined
let person = { name: "Bob", age: 40 }; // Object

Fun facts:

  • In JavaScript, "5" (a string) is different from 5 (a number).

  • Null and undefined are different even though both mean "nothing".

Operators allow you to do things with variables and values.

Arithmetic Operators:

4. Template Literals

Template literals make it easy to build strings with variables inside them.

  • Enclosed by backticks ` ` (not quotes ' or ")

  • Insert variables using ${variable}

Example:

javascript
let name = "David";
let age = 25;

// Old way
let message1 = "My name is " + name + " and I am " + age + " years old.";

// Modern way (Template Literals)
let message2 = `My name is ${name} and I am ${age} years old.`;

console.log(message1);
console.log(message2);

Template literals are cleaner, easier to read, and better for long strings!

5. String Methods

JavaScript gives you many built-in methods to work with strings.

MethodPurposeExample
lengthNumber of characters"hello".length // 5
toUpperCase()Convert to uppercase"hello".toUpperCase() // "HELLO"
toLowerCase()Convert to lowercase"HELLO".toLowerCase() // "hello"
includes()Check if a string contains another string"hello".includes("ll") // true
slice(start, end)Extract part of the string"hello".slice(1, 4) // "ell"
replace(old, new)Replace part of a string"hello".replace("he", "ye") // "yello"

Example:

javascript
let text = "JavaScript is fun!";
console.log(text.length);          // 17
console.log(text.toUpperCase());   // "JAVASCRIPT IS FUN!"
console.log(text.includes("fun")); // true
console.log(text.slice(0, 10));    // "JavaScript"

Module 3: Control Flow in JavaScript

1. What is Control Flow?

  • Control Flow means how your program makes decisions and repeats actions.

  • In JavaScript, code usually runs from top to bottom, but you can control it using:

    • if, else if, else

    • switch

    • for loops

    • while loops

    • do...while loops

Control Flow allows you to make programs dynamic, smart, and responsive!

2. Conditional Statements (if, else if, else)

Conditional statements let you run different blocks of code based on conditions.

if statement

javascript
if (condition) {
  // code to run if condition is true
}

Example:

javascript
let age = 20;

if (age >= 18) {
  console.log("You are an adult.");
}

Explanation:

  • If age is 18 or more, it prints "You are an adult."

if...else statement

javascript
if (condition) {
  // code if condition is true
} else {
  // code if condition is false
}

Example:

javascript
let age = 16;

if (age >= 18) {
  console.log("You are an adult.");
} else {
  console.log("You are a minor.");
}

if...else if...else chain

Sometimes you have multiple conditions:

javascript
if (condition1) {
  // do something
} else if (condition2) {
  // do something else
} else {
  // do another thing
}

Example:

javascript
let score = 75;

if (score >= 90) {
  console.log("Grade: A");
} else if (score >= 80) {
  console.log("Grade: B");
} else if (score >= 70) {
  console.log("Grade: C");
} else {
  console.log("Grade: Fail");
}

3. Switch Statements

Sometimes using many if/else can be messy.
When you have many options based on the same variable, use switch.

switch statement

javascript
switch(expression) {
  case value1:
    // code to run if expression === value1
    break;
  case value2:
    // code to run if expression === value2
    break;
  default:
    // code if no case matches
}

Example:

javascript
let day = 3;

switch(day) {
  case 1:
    console.log("Monday");
    break;
  case 2:
    console.log("Tuesday");
    break;
  case 3:
    console.log("Wednesday");
    break;
  default:
    console.log("Another day");
}

Important:

  • Always use break; after each case, or it will continue running next cases (called "fall-through").

4. Loops

Loops let you repeat actions multiple times without rewriting your code.

for loop

for loops run a block a specific number of times.

javascript
for (initialization; condition; increment) {
  // code to run each time
}

Example:

javascript
for (let i = 1; i <= 5; i++) {
  console.log("Number: " + i);
}

Explanation:

  • Start at i = 1

  • While i <= 5, run the block

  • Increase i by 1 after each run

while loop

Runs a block as long as the condition is true.

javascript
while (condition) {
  // code
}

Example:

javascript
let i = 1;

while (i <= 5) {
  console.log("While loop number: " + i);
  i++;
}

Use while when you don't know exactly how many times you need to loop.

do...while loop

  • do...while will always run at least once, then check the condition.

javascript
do {
  // code
} while (condition);

Example:

javascript
let i = 1;

do {
  console.log("Do...While loop number: " + i);
  i++;
} while (i <= 5);

5. Break and Continue

break

  • Stops the loop completely.

continue

  • Skips to the next loop iteration.

Example of break:

javascript
for (let i = 1; i <= 5; i++) {
  if (i === 3) break;
  console.log(i);
}
// Output: 1 2

Example of continue:

javascript
for (let i = 1; i <= 5; i++) {
  if (i === 3) continue;
  console.log(i);
}
// Output: 1 2 4 5

Module 4: Functions and Scope

1. What is a Function?

  • A function is a block of code designed to perform a particular task.

  • Functions help you organize, reuse, and manage code easily.

Think of a function like a machine:
You give it input (parameters) it processes and it gives you output.

2. Creating Functions

There are two main ways to create functions:

Function Declaration

javascript
function functionName(parameters) {
  // code to be executed
}

Example:

javascript
function greet(name) {
  console.log("Hello, " + name + "!");
}

greet("David");

Output:
Hello, David!

Function Expression

Functions can also be stored in a variable.

javascript
const functionName = function(parameters) {
  // code to be executed
};

Example:

javascript
const greet = function(name) {
  console.log("Hello, " + name + "!");
};

greet("Sarah");

Output:
Hello, Sarah!

Key difference:

  • Function Declarations are hoisted (you can call them before defining).

  • Function Expressions are not hoisted (you must define before using).

3. Parameters and Arguments

  • Parameters are placeholders inside the function definition.

  • Arguments are the actual values you pass when calling the function.

Example:

javascript
function add(x, y) { // x and y are parameters
  console.log(x + y);
}

add(5, 3); // 5 and 3 are arguments

Output:
8

4. Return Values

Functions can return a value using the return keyword.

  • After return, the function stops executing.

Example:

javascript
function multiply(x, y) {
  return x * y;
}

let result = multiply(4, 5);
console.log(result);

Output:
20

5. Arrow Functions (ES6)

Shorter way to write functions introduced in ES6 (modern JavaScript).

Syntax:

javascript
const functionName = (parameters) => {
  // code
};

If the function has one line, you can omit {} and return:

javascript
const add = (x, y) => x + y;

Example:

javascript
const greet = (name) => console.log("Hello, " + name + "!");
greet("Mugabi");

Output:
Hello, Mugabi!

6. Function Scope

Scope = where variables are accessible.

Local Scope (inside functions)

Variables declared inside a function are not accessible outside.

Example:

javascript
function sayHello() {
  let message = "Hello there!";
  console.log(message);
}

sayHello();
// console.log(message); //  Error: message is not defined

Global Scope

Variables declared outside functions are accessible everywhere.

Example:

javascript
let globalVar = "I'm global!";

function printGlobal() {
  console.log(globalVar);
}

printGlobal();
console.log(globalVar);

Both lines will print:
I'm global!

7. Block Scope (let and const)

  • let and const are block-scoped.

  • var is function-scoped (old style, avoid it).

Always use let or const.

Example:

javascript
{
  let x = 10;
  const y = 20;
  // x and y exist only inside this block
}
// console.log(x); //  Error

8. Default Parameters

You can set default values for function parameters.

Example:

javascript
function greet(name = "Guest") {
  console.log("Hello, " + name + "!");
}

greet(); // Hello, Guest!
greet("James"); // Hello, James!

Module 5: Arrays and Array Methods

1. What is an Array?

  • An array is a special variable that can hold more than one value at a time.

  • Arrays store lists of data (e.g., list of names, scores, products).

Think of an array like a container that organizes multiple items inside it.

Example:

javascript
let fruits = ["Apple", "Banana", "Mango", "Orange"];

fruits array has 4 items.

2. Creating Arrays

You can create arrays in different ways:

Method 1: Using Square Brackets (Recommended)

javascript
let colors = ["Red", "Green", "Blue"];

Method 2: Using the Array Constructor

javascript
let colors = new Array("Red", "Green", "Blue"); // Not recommended usually

Best practice: Use square brackets [].

3. Accessing Elements

  • Each item in an array has a position number, called an index.

  • Index starts from 0 (zero-based indexing).

Example:

javascript
let fruits = ["Apple", "Banana", "Mango", "Orange"];

console.log(fruits[0]); // Apple
console.log(fruits[2]); // Mango

4. Changing Elements

You can change an item in an array by accessing its index.

Example:

javascript
let fruits = ["Apple", "Banana", "Mango"];

fruits[1] = "Kiwi";

console.log(fruits); // ["Apple", "Kiwi", "Mango"]

5. Array Length

  • .length tells you how many items are in the array.

Example:

javascript
let fruits = ["Apple", "Banana", "Mango"];

console.log(fruits.length); // 3

Important: The highest index = length - 1.

6. Adding and Removing Items

push() - Add to the end

javascript
fruits.push("Pineapple");

Adds "Pineapple" at the end.

pop() - Remove from the end

javascript
fruits.pop();

Removes the last item.

unshift() - Add to the beginning

javascript
fruits.unshift("Strawberry");

Adds "Strawberry" at the start.

shift() - Remove from the beginning

javascript
fruits.shift();

Removes the first item.

7. Important Array Methods

These are very important for interviews and React!

indexOf()

Finds the position (index) of an item.

javascript
let fruits = ["Apple", "Banana", "Mango"];

console.log(fruits.indexOf("Banana")); // 1

If the item doesnt exist, it returns -1.

includes()

Checks if an item exists in the array.

javascript
console.log(fruits.includes("Mango")); // true
console.log(fruits.includes("Orange")); // false

slice()

Creates a copy of part of the array.

javascript
let fruits = ["Apple", "Banana", "Mango", "Orange"];

let someFruits = fruits.slice(1, 3);

console.log(someFruits); // ["Banana", "Mango"]

slice(startIndex, endIndex) (endIndex is not included).

splice()

Changes an array by removing or replacing elements.

javascript
let fruits = ["Apple", "Banana", "Mango"];

fruits.splice(1, 1, "Kiwi");

console.log(fruits); // ["Apple", "Kiwi", "Mango"]

splice(start, howMany, itemToAdd)

join()

Joins all array elements into a single string.

javascript
let fruits = ["Apple", "Banana", "Mango"];

console.log(fruits.join(", ")); // "Apple, Banana, Mango"

reverse()

Reverses the array in place.

javascript
let fruits = ["Apple", "Banana", "Mango"];

fruits.reverse();

console.log(fruits); // ["Mango", "Banana", "Apple"]

sort()

Sorts array alphabetically (for strings).

javascript
let fruits = ["Banana", "Apple", "Mango"];

fruits.sort();

console.log(fruits); // ["Apple", "Banana", "Mango"]

Caution:
For numbers, sort() can behave strangely you'll later learn about custom sort functions.

8. Looping Through Arrays

Often you want to do something with every item.

Using for loop

javascript
let fruits = ["Apple", "Banana", "Mango"];

for (let i = 0; i < fruits.length; i++) {
  console.log(fruits[i]);
}

Using for...of loop

javascript
for (let fruit of fruits) {
  console.log(fruit);
}

for...of is cleaner and more modern.

Module 6: Objects in JavaScript

1. What is an Object?

  • An object is a collection of properties.

  • A property is a key-value pair.

Objects are used to store related data and functions together.

Think of an object like a real-world object:
Example: a Car has:

  • brand: "Toyota"

  • color: "White"

  • speed: 180km/h

All these belong together in one object.

Example:

javascript
let car = {
  brand: "Toyota",
  color: "White",

};

2. Creating Objects

There are two common ways:

Method 1: Object Literal (Recommended)

javascript
let person = {
  name: "John",
  age: 25,
  isStudent: true
};

Method 2: Using new Object()

javascript
let person = new Object();
person.name = "John";
person.age = 25;
person.isStudent = true;

Best practice: Use Object Literal {} because it's cleaner.

3. Accessing Object Properties

You can access properties in two ways:

1 Dot Notation

javascript
console.log(person.name);  // John
console.log(person.age);   // 25

This is the most common and easy method.

2 Bracket Notation

javascript
console.log(person["name"]);  // John
console.log(person["age"]);   // 25

Bracket notation is useful when:

  • The key is stored in a variable.

  • The key has special characters.

4. Updating and Adding Properties

Update an existing property:

javascript
person.age = 26;

Add a new property:

javascript
person.country = "Uganda";

Objects are dynamic you can add new properties anytime!

5. Deleting Properties

Remove a property using delete:

javascript

console.log(person);

6. Nesting (Objects Inside Objects)

You can put an object inside another object.

javascript
let student = {
  name: "Alice",
  age: 22,
  address: {
    city: "Kampala",
    country: "Uganda"
  }
};

console.log(student.address.city); // Kampala

7. Methods in Objects

An object method is a function inside an object.

javascript
let person = {
  name: "John",
  greet: function() {
    console.log("Hello, my name is " + this.name);
  }
};

person.greet(); // Hello, my name is John

this refers to the current object.

8. Object.keys(), Object.values(), Object.entries()

These are very useful when working with objects.

Object.keys()

Returns an array of property names (keys):

javascript
let person = { name: "John", age: 25 };

console.log(Object.keys(person)); 
// ["name", "age"]

Object.values()

Returns an array of property values:

javascript
console.log(Object.values(person)); 
// ["John", 25]

Object.entries()

Returns an array of key-value pairs:

javascript
console.log(Object.entries(person)); 
// [["name", "John"], ["age", 25]]

9. Looping Through Objects

Use for...in to loop through properties:

javascript
let person = { name: "John", age: 25 };

for (let key in person) {
  console.log(key + ": " + person[key]);
}

key gives you the property name.
person[key] gives you the value.
