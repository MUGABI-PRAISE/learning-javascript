📘 Topic: async/await

✨ 1. What is async/await in JavaScript?

  • async/await is syntax sugar built on top of Promises.

  • It makes writing asynchronous (async) code look synchronous and easier to understand.

  • async: a keyword to declare that a function is asynchronous and will return a Promise.

  • await: pauses the execution inside an async function until the Promise is resolved or rejected.

✨ 2. Why use async/await?

  • It simplifies chaining .then() and .catch() from Promises.

  • Error handling becomes easier with try-catch blocks.

  • Your code becomes cleaner and more readable.

✨ 3. How to use async/await? (Syntax)

javascript
async function fetchData() {
  try {
    const response = await fetch('https://jsonplaceholder.typicode.com/posts');
    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.error('Error fetching data:', error);
  }
}
fetchData();

✅ Explanation:

  • async function fetchData() — declares an asynchronous function.

  • await fetch(...) — waits for the fetch Promise to resolve.

  • await response.json() — waits for the body to be parsed as JSON.

  • try...catch — standard error handling around async operations.

✨ 4. Important rules for using async/await

  • await can only be used inside an async function.

  • If you await a non-Promise value, it automatically wraps it into a Promise.

Example:

javascript
async function example() {
  return "Hello";
}

example().then(console.log); // "Hello"

Even though example() just returns a string, calling it gives a Promise automatically.

📘 Topic: JSON (stringify and parse)

✨ 1. What is JSON?

  • JSON stands for JavaScript Object Notation.

  • It's a lightweight data format used to store and exchange data.

  • JSON is basically a string that looks like a JavaScript object.

Example of JSON string:

json
{
  "name": "John",
  "age": 30,
  "isStudent": false
}

Notice:

  • All keys and string values must be inside double quotes ".

  • No trailing commas!

✨ 2. Why do we use JSON?

  • To send data from a client (browser) to a server and vice versa.

  • APIs usually send responses in JSON format.

  • You can't send JavaScript objects directly over the network — you must convert them to JSON strings.

✨ 3. JSON.stringify()

  • Converts a JavaScript object into a JSON string.

Syntax:

javascript
const jsonString = JSON.stringify(object);

Example:

javascript
const user = { name: "Alice", age: 25 };
const userJSON = JSON.stringify(user);

console.log(userJSON);
// Output: '{"name":"Alice","age":25}'

✅ Now it's a string you can send over the network!

✨ 4. JSON.parse()

  • Converts a JSON string back into a JavaScript object.

Syntax:

javascript
const object = JSON.parse(jsonString);

Example:

javascript
const jsonString = '{"name":"Alice","age":25}';
const user = JSON.parse(jsonString);

console.log(user.name); // Output: Alice

✅ Now it's a regular JavaScript object again!

✨ 5. Common Use-Case: Working with LocalStorage

  • LocalStorage can only store strings.

  • So you stringify() objects before storing.

  • And you parse() them after retrieving.

Example:

javascript
// Save to LocalStorage
const user = { name: "Bob", loggedIn: true };
localStorage.setItem('user', JSON.stringify(user));

// Retrieve from LocalStorage
const retrievedUser = JSON.parse(localStorage.getItem('user'));
console.log(retrievedUser.name); // Bob

📘 Module 8: Object-Oriented Programming (OOP) Basics in JS

✨ 1. What is Object-Oriented Programming (OOP)?

  • OOP is a programming style where we organize code using objects.

  • Objects have:

    • Properties (data).

    • Methods (functions that belong to the object).

In short:
OOP is about modeling real-world things as code.

Example:

javascript
const car = {
  brand: "Toyota",
  start: function() {
    console.log("Car started");
  }
};

car.start(); // Car started

✨ 2. Constructor Functions (Old way to create objects)

Before ES6 (2015), we used constructor functions to make many similar objects.

✅ A constructor function is:

  • A regular function.

  • Capitalized (by convention).

  • Used with the new keyword.

🔥 Syntax:

javascript
function Person(name, age) {
  this.name = name;
  this.age = age;
  this.greet = function() {
    console.log(`Hello, my name is ${this.name}`);
  };
}

// Creating an object
const person1 = new Person("Alice", 25);
const person2 = new Person("Bob", 30);

person1.greet(); // Hello, my name is Alice
person2.greet(); // Hello, my name is Bob

✅ Explanation:

  • this.name = name: adds a property to the new object.

  • new Person():

    • Creates a new object.

    • Sets this to that new object.

    • Returns the object automatically.

✨ 3. Classes (Modern way)

  • Introduced in ES6.

  • Cleaner, more intuitive syntax.

  • Under the hood, still based on prototypes.

🔥 Syntax:

javascript
class Person {
  constructor(name, age) {
    this.name = name;
    this.age = age;
  }

  greet() {
    console.log(`Hi, I am ${this.name}`);
  }
}

// Creating an object
const person1 = new Person("Charlie", 28);
person1.greet(); // Hi, I am Charlie

✅ Key things:

  • constructor() is a special method that runs when you create a new object.

  • Methods like greet() are automatically added to the prototype — efficient!

✨ 4. Inheritance (Simple understanding)

Inheritance = One class inherits from another, getting all its properties and methods.

✅ In JavaScript, we use extends keyword.

🔥 Syntax:

javascript
class Animal {
  constructor(name) {
    this.name = name;
  }

  speak() {
    console.log(`${this.name} makes a noise.`);
  }
}

class Dog extends Animal {
  speak() {
    console.log(`${this.name} barks.`);
  }
}

const dog1 = new Dog("Rex");
dog1.speak(); // Rex barks.

✅ Explanation:

  • Dog inherits from Animal.

  • Dog gets all of Animal's features.

  • We can override methods (like speak()).

📘 Module 9: LocalStorage and SessionStorage

✨ 1. What is Web Storage?

Web browsers give us storage space to save data locally in the user's browser.
There are two main types:

TypeLifespanPurpose
LocalStorageStays until manually clearedSave long-term data
SessionStorageCleared when tab/window closesSave short-term data for a session

✅ Both store only string data.
✅ To save objects or arrays, use JSON.stringify() and JSON.parse() (like we learned earlier).

✨ 2. LocalStorage Basics

  • Persistent storage — survives browser restarts.

  • Stored as key-value pairs (both are strings).

🔥 Common Methods:

MethodDescription
localStorage.setItem(key, value)Save a value
localStorage.getItem(key)Retrieve a value
localStorage.removeItem(key)Delete a value
localStorage.clear()Delete everything

🔥 Example:

javascript
// Save data
localStorage.setItem('username', 'Alice');

// Retrieve data
const user = localStorage.getItem('username');
console.log(user); // Alice

// Remove specific item
localStorage.removeItem('username');

// Clear all data
localStorage.clear();

✅ Important:

  • LocalStorage is shared between all tabs/windows from the same origin (same site + protocol).

  • No expiry date unless you manually remove it!

✨ 3. SessionStorage Basics

  • Same API as LocalStorage.

  • Data exists only until the tab is closed.

🔥 Example:

javascript
// Save data
sessionStorage.setItem('sessionId', '12345');

// Retrieve data
const sessionId = sessionStorage.getItem('sessionId');
console.log(sessionId); // 12345

// Remove item
sessionStorage.removeItem('sessionId');

// Clear all session data
sessionStorage.clear();

✨ 4. Storing Objects (Remember!)

Since Storage only saves strings, you must use:

javascript
// Saving an object
const user = { name: "Alice", loggedIn: true };
localStorage.setItem('user', JSON.stringify(user));

// Retrieving the object
const retrievedUser = JSON.parse(localStorage.getItem('user'));
console.log(retrievedUser.name); // Alice

✅ Always stringify before saving and parse after retrieving!

✨ 5. JavaScript (app.js)

Now for the real action!

javascript
// Select elements
const taskInput = document.getElementById('taskInput');
const addBtn = document.getElementById('addBtn');
const taskList = document.getElementById('taskList');

// Load tasks from LocalStorage
let tasks = JSON.parse(localStorage.getItem('tasks')) || [];

// Display tasks on page load
renderTasks();

// Add Task
addBtn.addEventListener('click', () => {
  const taskText = taskInput.value.trim();
  if (taskText !== "") {
    const task = { text: taskText, completed: false };
    tasks.push(task);
    saveTasks();
    renderTasks();
    taskInput.value = "";
  }
});

// Render tasks to the DOM
function renderTasks() {
  taskList.innerHTML = "";
  tasks.forEach((task, index) => {
    const li = document.createElement('li');
    li.textContent = task.text;
    if (task.completed) {
      li.classList.add('completed');
    }
    
    // Toggle Complete on Click
    li.addEventListener('click', () => {
      tasks[index].completed = !tasks[index].completed;
      saveTasks();
      renderTasks();
    });

    // Delete Button
    const deleteBtn = document.createElement('button');
    deleteBtn.textContent = "❌";
    deleteBtn.addEventListener('click', (e) => {
      e.stopPropagation(); // Prevent triggering the toggle
      tasks.splice(index, 1);
      saveTasks();
      renderTasks();
    });

    li.appendChild(deleteBtn);
    taskList.appendChild(li);
  });
}

// Save tasks to LocalStorage
function saveTasks() {
  localStorage.setItem('tasks', JSON.stringify(tasks));
}
