Hoisting (Variables and Functions)

What is Hoisting?

Hoisting is a JavaScript behavior where variable and function declarations are moved ("hoisted") to the top of their containing scope (function or global) during the compilation phase, before the code is executed.

Important:

  • Only the declaration is hoisted.

  • Initialization/assignment stays in place.

1. Hoisting with Variables

There are three ways to declare variables in JavaScript:

  • var

  • let

  • const

var is hoisted differently than let and const.

Example with var:

javascript
console.log(a); // Output: undefined
var a = 5;
console.log(a); // Output: 5

What happens behind the scenes:

javascript
var a;           // Declaration is hoisted
console.log(a);  // undefined (because only declaration, no assignment yet)
a = 5;           // Assignment happens here
console.log(a);  // 5

Example with let and const:

javascript
console.log(b); //  ReferenceError: Cannot access 'b' before initialization
let b = 10;

console.log(c); //  ReferenceError: Cannot access 'c' before initialization
const c = 15;

What happens:

  • let and const are also hoisted, but they stay in a "Temporal Dead Zone" (TDZ).

  • You can't use them before the line where they are declared.

Quick summary:

KeywordHoisted?Usable Before Declaration?Default Value if accessed early
varYesYes (but undefined)undefined
letYesNo (ReferenceError)N/A
constYesNo (ReferenceError)N/A

2. Hoisting with Functions

There are two ways to define functions:

  • Function Declarations

  • Function Expressions

Function Declarations:

Fully hoisted.

Example:

javascript
greet(); // Output: Hello!

function greet() {
  console.log('Hello!');
}

Behind the scenes, JavaScript hoists the whole function to the top.

Function Expressions:

Not fully hoisted (works like variables).

Example:

javascript
sayHi(); //  TypeError: sayHi is not a function

var sayHi = function() {
  console.log('Hi!');
};

  • sayHi is hoisted as a variable (undefined at first).

  • You can't call it before it's assigned the function.

Quick summary for functions:

TypeHoisted?Usable Before Declaration?
Function DeclarationYesYes
Function ExpressionNo (only variable name hoisted)No

Important Takeaways:

  • Declarations are hoisted, assignments are not.

  • var variables are hoisted and initialized with undefined.

  • let and const variables are hoisted but in a TDZ (Temporal Dead Zone) accessing them early throws an error.

  • Function declarations are fully hoisted.

  • Function expressions are treated like variables (var, let, const).

Closures (Basic Idea, Not Too Deep Yet)

What is a Closure?

A closure is simply when a function remembers the variables from the place where it was created, even if that function is used somewhere else.

In simple words:

  • A function "remembers" its surrounding variables even after the outer function has finished running.

Real Simple Example

javascript
function outer() {
  let count = 0;

  function inner() {
    console.log(count);
  }

  return inner;
}

const myFunc = outer();  // outer() runs and returns inner
myFunc();                // Output: 0

What happened here?

  • outer() creates a variable count.

  • inner() uses count.

  • Even though outer() has finished running, myFunc() still remembers count because of closure!

Why Are Closures Useful?

Closures allow us to:

  • Remember data (variables) between function calls.

  • Create private variables (variables that only the inner function can access).

  • Create powerful patterns like function factories, modules, and callbacks.

Closures are everywhere in JavaScript especially when you work with:

  • Event Listeners

  • Callbacks

  • Timers

  • React Components (Hooks are built on closures!)

Let's Build Step-by-Step

Example 1: Closure with Counter

javascript
function createCounter() {
  let count = 0;

  return function() {
    count++;
    console.log(count);
  }
}

const counter1 = createCounter();
counter1(); // 1
counter1(); // 2
counter1(); // 3

const counter2 = createCounter();
counter2(); // 1

Explanation:

  • createCounter() creates a new count each time.

  • counter1 and counter2 have their own private count.

  • They don't interfere with each other.

  • The inner function remembers count because of closure.

Example 2: Closure in setTimeout

javascript
function greet(name) {
  setTimeout(function() {
    console.log('Hello, ' + name);
  }, 1000);
}

greet('Alice'); 
// After 1 second: "Hello, Alice"

Explanation:

  • setTimeout's callback function remembers name.

  • Even after greet has finished running, the name value is kept alive because of the closure!

Common Mistake to Know

If you don't understand closures, you may think the variables are "lost" but closures preserve them!

Bad thinking :

"Outer function is done, variables are gone."

Correct thinking :

"Inner function locks those variables in memory."

Very Important Key Points (to memorize):

Key IdeaExplanation
Closure = function + surrounding memoryA function keeps access to the variables around it.
Closures happen automaticallyYou don't need to do anything special.
Variables stay alive as long as neededEven after outer functions finish.
Used in real-world code a lotEvent listeners, timers, React hooks, API calls, etc.

Callback Functions (Explained Clearly and Simply)

What is a Callback Function?

Definition:
A callback is just a function passed into another function as an argument, and its called later inside that other function.

In short:

You give a function to another function, and the second one decides when to run it.

Why Do We Need Callbacks?

JavaScript is asynchronous things like:

  • Fetching data from the Internet

  • Waiting for a user click

  • Waiting for a timer to finish

These take time, and JavaScript doesn't like waiting (it continues running).
Callbacks help us say what to do after something happens.

Basic Example

javascript
function greet(name) {
  console.log("Hello, " + name);
}

function processUserInput(callback) {
  const name = "Alice"; // pretend we got user input
  callback(name);
}

processUserInput(greet);

Explanation:

  • greet is a function.

  • We pass greet into processUserInput as callback.

  • processUserInput calls the callback later with "Alice".

Another Example Using a Callback for Math

javascript
function add(a, b) {
  return a + b;
}

function calculate(a, b, callback) {
  return callback(a, b);
}

console.log(calculate(5, 3, add)); // Output: 8

Explanation:

  • calculate doesn't know how to calculate.

  • It trusts the callback to tell it what to do.

  • Here, we pass add as the callback.

Real-world Example setTimeout (Delay something)

javascript
setTimeout(function() {
  console.log("I show up after 2 seconds!");
}, 2000);

Explanation:

  • setTimeout takes two parameters:

    • a function (callback)

    • a time in milliseconds

  • After 2 seconds, the callback runs.

Important: Anonymous Functions as Callbacks

Instead of defining a named function, you can just pass a function directly.

Example:

javascript
processUserInput(function(name) {
  console.log("Hi there, " + name);
});

Here, we didn't even name the callback! (This is called an anonymous function.)

Promises in JavaScript (Beginner Friendly Guide)

What is a Promise?

Definition:
A Promise is a special JavaScript object that represents a value that you will get now, later, or never.

In other words:

A Promise is like a guarantee that something will happen in the future (success or failure).

Why Do We Need Promises?

Before Promises, JavaScript used only callbacks.
But callbacks caused "callback hell" messy, confusing code.

Promises make:

  • Asynchronous code cleaner

  • Easier to handle success and errors

  • Easier to chain multiple async operations

Promise States

Every Promise has 3 possible states:

StateMeaning
PendingThe promise is still waiting (not finished yet)
FulfilledThe promise completed successfully (you get the result)
RejectedThe promise failed (you get an error)

Basic Promise Example

javascript
const myPromise = new Promise(function(resolve, reject) {
  let success = true; // try changing to false to test failure

  if (success) {
    resolve("Promise was successful!");
  } else {
    reject("Promise failed!");
  }
});

myPromise
  .then(function(result) {
    console.log(result);
  })
  .catch(function(error) {
    console.log(error);
  });

Explanation:

  • new Promise takes a function with two arguments: resolve and reject.

  • Call resolve(result) when successful.

  • Call reject(error) when something goes wrong.

  • .then() runs if its successful.

  • .catch() runs if it fails.

Very Simple Example with Delay

javascript
const delayedPromise = new Promise(function(resolve, reject) {
  setTimeout(function() {
    resolve("Done after 2 seconds!");
  }, 2000);
});

delayedPromise
  .then(function(data) {
    console.log(data);
  });

Output:

nginx
Done after 2 seconds!

Explanation:

  • After 2 seconds, the Promise is resolved.

  • .then() runs when its ready.

Task 1: Create a Promise that resolves with "Data loaded" after 1 second.

javascript
const loadData = new Promise(function(resolve, reject) {
  setTimeout(function() {
    resolve("Data loaded");
  }, 1000); // 1 second
});

loadData
  .then(function(result) {
    console.log(result); // Output: Data loaded
  });

Explanation:

  • After 1 second, resolve is called.

  • .then() catches the success and logs "Data loaded".

Task 3: Create a function simulateRequest(success).

javascript
function simulateRequest(success) {
  return new Promise(function(resolve, reject) {
    if (success) {
      resolve("Request successful!");
    } else {
      reject("Request failed!");
    }
  });
}

// Test with success = true
simulateRequest(true)
  .then(function(result) {
    console.log("Result:", result);
  })
  .catch(function(error) {
    console.log("Error:", error);
  });

// Test with success = false
simulateRequest(false)
  .then(function(result) {
    console.log("Result:", result);
  })
  .catch(function(error) {
    console.log("Error:", error);
  });

Explanation:

  • simulateRequest(true) goes to .then().

  • simulateRequest(false) goes to .catch().

Small Real-World Example: Fetching API Data Using Promises

Let's use fetch() a built-in JavaScript function that returns a Promise automatically.

javascript
fetch("https://jsonplaceholder.typicode.com/posts/1")
  .then(function(response) {
    return response.json(); // parsing JSON
  })
  .then(function(data) {
    console.log("Post Title:", data.title);
  })
  .catch(function(error) {
    console.log("Fetch error:", error);
  });

Explanation:

  • fetch gets data from the Internet.

  • It returns a Promise.

  • First .then() reads the response.

  • Second .then() gets the data.

  • .catch() handles any error.
